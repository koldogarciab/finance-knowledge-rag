# Local LLM generation with Ollama

The project can generate grounded answers locally without an external API,
API key or token charges.

## Tested configuration

- Operating system: Windows
- Ollama: 0.31.1
- Model: `qwen3.5:4b`
- Model size: approximately 3.4 GB
- Execution: CPU
- Local API: `http://localhost:11434`

## Install Ollama

Install Ollama using the official Windows installer or PowerShell installer:

```powershell
irm https://ollama.com/install.ps1 | iex
```

Close and reopen PowerShell, then verify the installation:

```powershell
ollama --version
```

## Download the local model

```powershell
ollama pull qwen3.5:4b
ollama list
```

## Test the model

```powershell
ollama run qwen3.5:4b "Explain gross margin in one clear sentence."
```

## Environment configuration

The following settings are optional because the application already uses
these defaults:

```text
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=qwen3.5:4b
OLLAMA_TIMEOUT_SECONDS=300
OLLAMA_TEMPERATURE=0
OLLAMA_NUM_PREDICT=512
OLLAMA_KEEP_ALIVE=5m
```

To override them, copy `.env.example` to `.env`. The `.env` file is excluded
from version control.

## Run the grounded local RAG

From the repository root:

```powershell
python -m src.ollama_rag "How is gross margin calculated and what target applied for the nine months ended 31 March 2026?" --top-k 5
```

The pipeline:

1. retrieves evidence using the hybrid dense and BM25 retriever;
2. checks whether the retrieved evidence supports the question;
3. sends only the selected evidence to the local model;
4. validates every citation generated by the model;
5. falls back to a deterministic grounded answer if generation fails;
6. abstains before generation when the corpus cannot support the answer.

## Automated validation

The automated tests use simulated Ollama responses. They do not require the
local service and do not load the language model:

```powershell
python tests\validate_ollama_adapter.py
python tests\validate_ollama_rag.py
```

A live Ollama execution is tested separately through the command-line
interface.
